# Project Scaffolding Skill

Create a new Claude project with a standardized directory structure on the NAS. This ensures all projects follow the same organization, making it easy to apply the project-status → git-checkin → social-post workflow to any project.

## When to use this skill

- **Starting a new project:** Before you begin any work
- **Consolidating scattered work:** Moving an existing project into the standard structure
- **User says:** "Create a new project", "Start a project", "Set up a project", "New project for X"

## Project Structure Created

Each project gets this standardized tree:

```
/Volumes/REP/projects/Claude/[project-name]/
├── README.md                    # Project overview template
├── PROJECT_STATUS.md            # (empty initially, generated by project-status skill)
├── .claude/
│   ├── settings.json            # Project-specific Claude settings
│   └── memory/
│       ├── MEMORY.md            # Memory index
│       ├── project-context.md   # Project goals, scope, stakeholders
│       └── [other memory files created as needed]
├── src/                         # Project source code/content
├── docs/                        # Documentation
├── social_media/
│   ├── POSTS.md                 # Archive of published posts
│   └── assets/                  # Images, screenshots for social posts
├── .gitignore                   # Standard git ignore rules
└── .git/                        # (optional, created if user wants git initialized)
```

## Instructions

1. **Ask for project details:**
   - Project name (e.g., "auth-redesign", "data-pipeline", "blog-rewrite")
   - Project type (optional, for informational purposes)
   - Should git be initialized? (yes/no)
   - Any specific technology stack or notes? (optional)

2. **Create directory structure:**
   ```bash
   mkdir -p /Volumes/REP/projects/Claude/[project-name]/{src,docs,social_media/assets,.claude/memory}
   ```

3. **Initialize template files:**

   **README.md:**
   ```markdown
   # [Project Name]
   
   **Status:** In progress
   **Started:** [YYYY-MM-DD]
   
   ## Overview
   [User describes what this project is about]
   
   ## Goals
   - [Primary goal]
   - [Secondary goals]
   
   ## Progress
   [Will be updated as project evolves]
   
   ---
   
   Created with the Claude Project Scaffold skill.
   ```

   **.claude/settings.json:**
   ```json
   {
     "project_name": "[project-name]",
     "project_type": "[user input or 'general']",
     "created": "[YYYY-MM-DD]",
     "location": "/Volumes/REP/projects/Claude/[project-name]/",
     "memory_enabled": true
   }
   ```

   **.claude/memory/MEMORY.md:**
   ```markdown
   # Memory Index
   
   - [project-context](project-context.md) — Project goals, scope, and context
   ```

   **.claude/memory/project-context.md:**
   ```markdown
   ---
   name: project-context
   description: Project goals, scope, stakeholders, and key constraints
   metadata:
     type: project
   ---
   
   # [Project Name] Context
   
   **Project type:** [User specified]
   **Started:** [YYYY-MM-DD]
   **Location:** /Volumes/REP/projects/Claude/[project-name]/
   
   ## Goals
   [User describes what they're trying to accomplish]
   
   ## Scope
   [What's in scope, what's out of scope]
   
   ## Stakeholders
   [Who cares about this project]
   
   ## Key Constraints
   [Budget, timeline, technical constraints, etc.]
   
   ## Success Criteria
   [How will we know this is done/successful?]
   ```

   **.gitignore** (standard template):
   ```
   # Dependencies
   node_modules/
   __pycache__/
   venv/
   .venv/
   
   # Environment variables
   .env
   .env.local
   .env.*.local
   
   # IDE/Editor
   .vscode/
   .idea/
   *.swp
   *.swo
   *~
   .DS_Store
   
   # Build/Dist
   build/
   dist/
   *.pyc
   *.egg-info/
   
   # OS
   .DS_Store
   Thumbs.db
   
   # Project specific
   # Add any project-specific patterns below
   ```

   **PROJECT_STATUS.md** (empty initially):
   ```markdown
   # Project Status Report
   [To be generated by project-status skill]
   ```

   **social_media/POSTS.md** (empty initially):
   ```markdown
   # Social Media Posts Archive
   [Posts will be archived here by social-post-blotato skill]
   ```

4. **Initialize git (if requested):**
   ```bash
   cd /Volumes/REP/projects/Claude/[project-name]/
   git init
   git add .
   git commit -m "chore: initial project scaffold"
   ```

5. **Report success:**
   Show the user:
   ```
   ✅ Project created: /Volumes/REP/projects/Claude/[project-name]/
   
   📁 Directory structure:
   ├── README.md (fill this in with project details)
   ├── PROJECT_STATUS.md (generated by project-status skill)
   ├── .claude/
   │   ├── settings.json
   │   └── memory/
   │       ├── MEMORY.md
   │       └── project-context.md (describe your project here)
   ├── src/
   ├── docs/
   ├── social_media/
   │   ├── POSTS.md
   │   └── assets/
   └── .gitignore
   
   🚀 Next steps:
   1. Edit README.md with your project overview
   2. Edit .claude/memory/project-context.md with goals and scope
   3. Start working in src/ or docs/
   4. End of session: Run project-workflow skill (status → commit → social posts)
   ```

## Benefits of standardization

- **Consistent organization** — All projects follow the same structure
- **Memory system ready** — Claude can build context for each project
- **Workflow compatible** — Works seamlessly with project-status, git-checkin, social-post-blotato
- **NAS centralized** — All projects on /Volumes/REP/projects/Claude/ (SSD stays clean)
- **Easy discovery** — Find any project in one place

## Edge cases

- **Project name conflicts** — If directory already exists, ask user: "Project exists. Overwrite? (y/n)" or suggest a new name
- **NAS unavailable** — If /Volumes/REP not mounted, warn user and stop
- **Special characters in name** — Sanitize project name (convert to lowercase, replace spaces with hyphens)
- **Existing git repo** — If user specifies git init and .git already exists, skip git init

## Integration with workflow

After scaffolding:
1. User edits README.md and project-context.md
2. User begins work on the project
3. At end of session, runs **project-workflow** skill:
   - project-status reads conversation → fills PROJECT_STATUS.md
   - git-checkin commits changes
   - social-post-blotato publishes highlights to social media

This skill sets up the foundation for that entire workflow.
